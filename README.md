# Система управления заказами «Быстрая доставка»

Внутреннее приложение для учёта заказов доставки, управления клиентской базой и построения аналитики. 

Приложение предоставляет два интерфейса:
- **Графический (GUI)** — для повседневной работы менеджеров.
- **Консольный (CLI)** — для административных задач и автоматизации.

## Основные возможности

- Полное управление клиентами (добавление, редактирование, удаление).
- Управление заказами с автоматическим расчётом итоговой суммы.
- Защита данных: невозможно удалить клиента, если у него есть заказы (ограничение БД ON DELETE RESTRICT).
- Экспорт и импорт данных в форматах JSON и XML.
- Построение отчётов (выручка за период, топ-3 клиента, статистика по статусам).
- Фильтрация заказов по статусу и дате.
- Логирование всех операций.

#### Технологический стек

- **Язык:** Python 3.8+
- **База данных:** SQLite3
- **GUI:** Tkinter
- **CLI:** argparse
- **Тесты:** pytest
- **Логирование:** logging

### Установка и запуск

1. Установите зависимости:
```bash
pip install -r requirements.txt
```

2. Запустите графический интерфейс:
```bash
python main_gui.py
```

4. Или используйте консольный интерфейс:
```bash
python main_cli.py --help
```

## Примеры использования CLI
Получить отчёт за месяц:
```bash
python main_cli.py report --period month
```
Добавить клиента:
```bash
python main_cli.py add-customer --name "Иван Иванов" --phone "+79991234567"
```

Экспорт и импорт заказов:
```bash
python main_cli.py export --file backup.json
python main_cli.py import --file backup.xml
```

## Структура проекта
- main_gui.py — графический интерфейс (Tkinter)
- main_cli.py — консольный интерфейс (argparse)
- database.py — слой работы с базой данных SQLite
- models.py — модели данных (Customer, Order, OrderItem)
- data_export.py — логика экспорта и импорта в JSON и XML
- logger_config.py — настройка логирования
- tests/ — папка с модульными тестами (pytest)
- data/ — папка для файла базы данных (создаётся автоматически при первом запуске)
- logs/ — папка для логов (создаётся автоматически при первом запуске)

## Тестирование
Для запуска тестов используйте команду:
```bash
pytest tests/ -v
```
Тесты покрывают работу с базой данных, экспорт/импорт и модели данных. Они используют изолированные базы данных в памяти и не влияют на основной файл delivery.db.

### Логирование
Все важные действия (создание, изменение, удаление записей, ошибки импорта) записываются в файл logs/app.log
